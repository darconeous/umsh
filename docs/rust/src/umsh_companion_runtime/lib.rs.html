1//! Board-agnostic companion-radio NCP runtime shared across firmware targets.
2//!
3//! Phase 5 extraction (see `docs/firmware-plan-heltec-lora32-v3.md`). This
4//! crate holds the parts of the companion NCP that have no board HAL
5//! dependency, so the nRF52840 (T-Echo / T-1000E) and ESP32-S3 (Heltec V3)
6//! firmwares consume one copy instead of maintaining divergent forks.
7//!
8//! Increment A moved the pure leaf modules: transport arbitration, the
9//! persisted counter map re-export, the BLE pairing-policy helpers, and the
10//! radio multiplexer. Increment C adds [`driver`]: the NCP session run loop
11//! (formerly the nRF `ncp_task`/`apply_effect`/`Emitter`), with every board
12//! coupling routed through the [`driver::NcpEnv`] trait.
13
