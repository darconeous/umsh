3//! Board support for the LilyGO T-Echo (nRF52840).
4//!
5//! Composes [`umsh-bsp-nrf52840`](../umsh_bsp_nrf52840/index.html) with
6//! the T-Echo's pinout and on-board peripherals.
7//!
8//! Initial scope is *bringup only* (see `docs/firmware-plan-techo.md`):
9//!
10//! - peripheral power switch (P0.12, active-high)
11//! - USB-CDC handles (over the nRF native USB peripheral)
12//! - blue LED (P0.14, active-low)
13//! - e-paper SPI bus + control pins for the SSD1681 display
14//!
15//! Future expansion (LoRa, GNSS, I²C peripherals, QSPI flash, buttons,
16//! Plus-only back-panel) is welcome but is not yet implemented. The
17//! T-Echo is not a tracker-class device — when a "real" T-Echo firmware
18//! arrives, it should be designed against a new `umsh-ux-handheld` (or
19//! similar) class rather than `umsh-ux-tracker`.
20//!
21//! See `docs/lilygo-techo-hardware.md` for the firmware-derived
22//! hardware reference.
