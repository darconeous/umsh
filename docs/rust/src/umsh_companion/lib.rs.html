3//! Wire format for the minimal UMSH companion-radio (NCP) protocol.
4//!
5//! This crate implements the byte-level grammar specified in
6//! `docs/protocol/src/companion-radio-minimal.md`: packed unsigned
7//! integers, the frame header and command layer, property/stream/status
8//! identifiers, the `STR_PHY_RAW` metadata envelopes, and HDLC-Lite
9//! framing for asynchronous serial transports (UART, USB-CDC).
10//!
11//! It is shared between the host side (a `umsh-hal::Radio`
12//! implementation that drives a companion radio over a serial link) and
13//! the NCP firmware side (the session that exposes the local radio).
14//! Every wire-format detail lives here and only here.
15//!
16//! The crate is `no_std`, allocation-free, and has no dependencies.
17//! Callers provide byte buffers; encoders return the number of bytes
18//! written and decoders borrow from the input.
