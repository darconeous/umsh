3//! NCP-side session engine for the minimal companion-radio protocol and
4//! independently advertised full-profile extensions.
5//!
6//! [`Session`] is a pure state machine: it consumes decoded
7//! companion-link frames (framing such as HDLC-Lite is handled by the
8//! caller), emits response frames through a caller-provided sink, and
9//! returns radio side effects ([`Effect`]) for the caller to execute.
10//! It performs no I/O and takes time as a parameter, so the whole
11//! protocol surface is testable on the host; firmware only supplies
12//! the byte pipes and the radio.
13//!
14//! Design constraints from the spec
15//! (`docs/protocol/src/companion-radio-minimal.md`):
16//!
17//! - one confirmed transmit in flight at a time (`STATUS_BUSY`);
18//! - transmits never wait for duty-cycle allowance
19//!   (`STATUS_DUTY_LIMIT` unless the `NODUTY` flag is set);
20//! - `CMD_RST` is a protocol-level reset: all session state returns to
21//!   post-reset defaults and the radio is re-applied, but the MCU (and
22//!   therefore the USB link) stays up.
23//!
24//! v0 limitations (all surfaced through defined status codes):
25//! `PROP_PHY_RSSI` is unimplemented, the sync word only accepts the
26//! value the firmware was built with, and the CCA transmit flag is
27//! ignored (frames transmit without a clear-channel check, matching
28//! the existing firmware radio path).
